# 版本管理说明

## Git 版本控制

本项目已使用 Git 进行版本控制，方便您管理代码版本和回退。

## 常用命令

### 查看当前状态
```bash
git status
```

### 查看提交历史
```bash
git log --oneline
```

### 查看所有版本标签
```bash
git tag
```

### 创建新版本标签
```bash
git tag -a v1.1 -m "版本说明"
```

### 切换到某个版本
```bash
# 切换到某个标签
git checkout v1.0

# 切换回最新版本
git checkout master
# 或
git checkout main
```

### 查看版本差异
```bash
git diff v1.0 HEAD
```

### 创建新提交保存当前更改
```bash
git add .
git commit -m "提交说明"
```

## 回退到之前的版本

### 方法1：查看并切换到之前的提交
```bash
# 查看提交历史
git log --oneline

# 切换到某个提交（查看模式，不修改文件）
git checkout <commit-hash>

# 如果想永久回退到某个版本
git reset --hard <commit-hash>
```

### 方法2：使用标签
```bash
# 切换到v1.0版本
git checkout v1.0

# 切换回最新版本
git checkout master
```

## 注意事项

⚠️ **重要提醒**：
- `git reset --hard` 会永久删除之后的提交，请谨慎使用
- 建议在回退前先创建当前版本的标签或备份
- 如果只是想查看旧版本，使用 `git checkout` 查看后记得切回 `master` 分支
